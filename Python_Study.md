## 1. 关于图论参数的“变与不变”

之前看基础模型都是 $a_{ij}=1$ 的无向图，但在实际车联网（选题A）里，情况复杂得多：

- **有向图**：这信息不是对等的，A 能看到 B，B 不一定能看到 A。公式里的邻接矩阵 $A$ 就不对称了。我查了一下，只要这个图里存在**生成树（Spanning Tree）**，也就是有一个“核心”能辐射到所有人，大家最后还是能达成一致的。
- **有权图**：$a_{ij}$ 不再是 0 或 1，而是代表了**信任度**或者**通信质量**。权重大的邻居，对我的状态影响就大。收敛速度快慢全看拉普拉斯矩阵的特征值。

## 2. 矩阵形式推导

把所有小车的状态排成一个大向量：
$$X = [x_1, x_2, \dots, x_n]^T$$
单个个体的逻辑是：
$$\dot{x}_i = \sum a_{ij} (x_j - x_i)$$
写成矩阵形式其实就是：
$$\dot{X} = -(D - A) X = -L X$$

这里的 $L$（拉普拉斯矩阵）简直是这个算法的灵魂。$D$ 是入度对角阵，$A$ 是邻接阵。**推导心得**：其实就是把求和号里的项拆开，发现正好符合矩阵乘法的行列对应关系。

## 3. 我们最终会“停”在哪？

- **理想状态**：如果是无向且连通的，大家最后会停在所有人的**初始平均值**上。
- **变数**：
  - **延时**：如果通信太慢，大家还没对齐就跑偏了，系统会**震荡**。
  - **偏差注入**：这就是选题 B 说的隐私攻击。如果有个“坏群之马”一直给邻居发错误数据，整个编队就会产生**稳态误差**，没法完美重合。

## 4. 离散 vs 连续：仿真时的坑

- **连续时间**：是理论上的理想态，$\dot{x}$ 是变化率，曲线很圆滑。
- **离散时间**：这是我们写代码（Python/C++）时真正用的。公式变成了 $x(k+1) = x(k) - \Delta t \cdot L x(k)$。
- **踩坑点**：步长 $\Delta t$ 不能乱设！如果设得太大，小车会直接飞出屏幕（发散）。
